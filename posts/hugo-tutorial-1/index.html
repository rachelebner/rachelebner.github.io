I tried to set up my own GitHub blog using jekyll and got stuck. I left it alone.

Then jumped on the opportunity to have a guided session wokring in it with the Tech Blogger’s community of “Baot IL”

I wrote this post as part of setting up the new blog,and also to document what I did, where I got stuck and how I overcame it

The tutorial

I followed the step in this tutorial:

https://gohugo.io/hosting-and-deployment/hosting-on-github/

The installation steps

Create a blog content locally

Here are the code pieces needed to set a little starter blog:

// create an empty dir
hugo new site quickstart

cd quickstart
git init
git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke

// add theme
echo 'theme = "ananke"' >> config.toml

// add a post
hugo new posts/my-first-post.md

Build and run locally

// run server with drafts:
hugo server -D

// build static pages into public directory:
hugo -D

// view here:
http://localhost:1313

Direct the public folder to push there

Step 6 is the most delicate one. The simple command is this

git submodule add -b master https://github.com/rachelebner/rachelebner.github.io.git public

If you get stuck and for some reason the public folder gets pushed to your project repo, try the following steps:

  1. Remove the submodule you created for public:
git submodule deinit public
git rm -r public
git commit -m "Removed submodule "
rm -rf .git/modules/public
pit push

  1. make sure the user repository has some content, e.g. a README file

  2. run the submodule creation again

git submodule add -b master https://github.com/rachelebner/rachelebner.github.io.git public

  1. go into public folder and pull You should see the readme file now
